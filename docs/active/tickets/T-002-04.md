## Context

Define the theme type system in moron-themes. Themes bridge Rust configuration to CSS custom properties consumed by the React rendering layer. The Theme struct defines colors, typography, spacing, and animation timing.

Define:
- `Theme` struct with fields for: name, colors (background, foreground, accent), typography (font families, sizes), spacing scale, animation timing
- `Theme::default()` returning the built-in dark theme (matching values from packages/themes/src/default.css)
- `ThemeColors`, `ThemeTypography`, `ThemeSpacing`, `ThemeTiming` sub-structs
- `Theme::to_css_properties() -> Vec<(String, String)>` method that serializes to CSS custom property pairs
- Serde Serialize/Deserialize derives for JSON serialization to the React layer

Files: moron-themes/src/theme.rs, moron-themes/src/defaults.rs, moron-themes/src/lib.rs

## Acceptance Criteria

- `Theme` struct with nested sub-structs defined
- `Theme::default()` returns sensible values matching default.css
- `to_css_properties()` returns correct CSS custom property pairs
- Serde serialization works
- `cargo check` passes
- Unit test: Theme::default() produces expected CSS properties
