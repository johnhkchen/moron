## Context

Running `cargo clippy -- -W clippy::pedantic` on moron-techniques reveals ~20 warnings including:

- Unreadable numeric literal `0.984375` needs underscores (`0.984_375`)
- `ease()` function missing `#[must_use]` attribute
- `fn name() -> &str` should return `&'static str` on Technique implementations
- Documentation missing backticks around type/function names

These are style improvements that bring moron-techniques up to pedantic clippy standards.

Relevant files:
- `moron-techniques/src/technique.rs`
- `moron-techniques/src/reveals.rs`
- `moron-techniques/src/motion.rs`
- `moron-techniques/src/lib.rs`

## Acceptance Criteria

- `cargo clippy -p moron-techniques -- -W clippy::pedantic` produces zero warnings
- No behavioral changes to any technique
- All existing tests pass unchanged
- `cargo clippy` (default lints) still clean across workspace
