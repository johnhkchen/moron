## Context

moron's current ElementKind variants are text-centric: Title, Section, Show, Metric, Steps. To support data visualizations (charts, maps, timelines), we need a way to pass arbitrary structured data from Rust scenes to React components.

A new `ElementKind::Data` variant carries a JSON payload and a component type identifier. The React renderer dispatches on the type identifier to render the appropriate custom component.

## Acceptance Criteria

- New `ElementKind::Data { component: String }` variant in frame.rs
- New facade method: `m.data(component: &str, payload: serde_json::Value) -> Element`
- The payload is stored as a JSON string in ElementRecord and passed through FrameState
- ElementState gains a `data` field: `Option<serde_json::Value>` (None for non-Data elements)
- Serialization: data field included in FrameState JSON, deserialized by React
- TypeScript types updated: `DataKind` variant with `component` and `data` fields
- MoronFrame renders Data elements with a placeholder ("Custom: {component}") until T-013-02 adds the registry
- Tests verify Data elements serialize correctly and survive round-trip
